Graph databases

A graph database is a systematic collection of data that emphasizes the relationship between the diffrent data enteties.

--> It is stored without restricting it to a pre-defined model so that there's a more flexible way of using it.

--> there are 2 popular models of graph databases:
Property graphs & RDF graphs

Both of these graph types consist of a collection of points (vertices) and the connection between those points (edges).

Property graphs: Property graphs are used to model relationships among data, and they enable query and data analytics based on these relationships. A property graph has vertices that can contain detailed information about a subject, and edges that denote the relationship between the vertices.

RDF graphs:RDF graphs (RDF stands for Rescource Description Framework) are best for representing complex metadata and masterdata. They are often used for linked data, data integration and knowledge graphs. They can represent complex concepts in a domain, or provide rich semantics and inferencing on data.
